# Test Map (Historical — Migration Planning Document)

> **This document is a historical planning artifact.** It was written before the processor refactoring to track which tests needed to be migrated, created, or updated. The refactoring described here is now complete. For the current, up-to-date test reference see [`tests/TESTS.md`](../tests/TESTS.md).

This document records what each test file checks and how it relates to the
processor refactoring.  Use it after the refactoring to verify that every
behavioural contract is still covered.

---

## Status legend

| Symbol | Meaning |
|--------|---------|
| ✅ CURRENT | Tests existing code.  Must stay green before, during, and after refactoring. |
| 🔴 TDD | Written as a specification for code that does not exist yet.  Expected to fail until the new architecture is implemented. |
| 🔁 MIGRATES | Tests current code that will move or be replaced; see the "After refactoring" column. |

## 2. Dataset → Processor contract

### `test_dataset_processor_contract.py` ✅ CURRENT
**Code under test:** `_generate_examples()` of each dataset class, using committed asset TSVs.

**Purpose:** Verifies that the dict yielded by the dataset contains *at least* the keys and value types that the processor regression-test fixtures assume.  This is the bridge between layer 1 (dataset) and layer 3 (processor).

| Class | TSV used | Keys verified |
|---|---|---|
| `TestPose2TextDatasetProcessorContract` | `tests/assets/pose/metadata.tsv` ¹ | `signal` (str, existing path), `signal_start` (int), `signal_end` (int), `encoder_prompt`, `decoder_prompt`, `output` |
| `TestVideo2TextDatasetProcessorContract` | `tests/assets/video/metadata.tsv` ¹ | same as pose |
| `TestFeatures2TextDatasetProcessorContract` | `tests/assets/features/metadata.tsv` ¹ | same as pose |
| `TestText2TextDatasetProcessorContract` | `tests/assets/text/metadata.tsv` (committed) | `signal`, `encoder_prompt`, `decoder_prompt`, `output` |
| `TestImage2TextDatasetProcessorContract` | `tests/assets/image/metadata.tsv` (committed) | same as text; also spot-checks exact string value |

¹ Generated by `tests/assets/generate_assets.py` (not committed — contains absolute paths).
Tests for these modalities skip automatically if the TSV is absent.

**After refactoring:** Must stay green.  If `MultimodalMetaProcessor` reads from different
column names, update both the dataset and this contract test to match.

## 4. DataCollator

### `test_datacollator.py` ✅ CURRENT  🔁 MIGRATES
**Code under test:** `create_seq2seq_labels_from_samples`, `DataCollatorMultimodalSeq2Seq`

| Class | What it checks |
|---|---|
| `TestCreateSeq2SeqLabels` | Label tensor shape; `-100` padding; EOS appended; `padding=False` returns raw lists; `max_length` and `pad_to_multiple_of`; missing output → `None` |
| `TestDataCollatorInit` | Tokenizer fallback to `processor.tokenizer` when `tokenizer=None` |
| `TestDataCollatorCall` | Full pipeline: `text2text` and `features2text` collator calls produce `input_*` + `labels`; batch dimension consistency |

**After refactoring:**
- `create_seq2seq_labels_from_samples` is removed from the DataCollator and replaced by
  `TextModalityProcessor(role="label").process_batch()`.
- `TestCreateSeq2SeqLabels` defines the exact behaviour that `TextModalityProcessor(role="label")`
  must reproduce (see also `TestLabelsRegression` for numerical equivalence).
- `DataCollatorMultimodalSeq2Seq` is simplified: it will no longer accept a `tokenizer`
  argument or call `create_seq2seq_labels_from_samples`.  `TestDataCollatorInit` and
  `TestDataCollatorCall` must be updated to match the new simplified signature.

## 5. Regression tests (golden values)

### `tests/assets/generate_golden.py`
Script that runs each processor (and `create_seq2seq_labels_from_samples`) with the
committed asset files and saves tensor statistics to `tests/assets/golden/*.json`.

Re-run after any **intentional** preprocessing change:
```bash
python tests/assets/generate_golden.py            # all modalities
python tests/assets/generate_golden.py pose text  # specific modalities
```

### `test_processor_regression.py` ✅ CURRENT  🔁 MIGRATES

**Purpose:** Verifies that the processor output tensors are numerically identical to the
committed golden values.  Catches unintended changes in preprocessing logic.

For float tensors: shape, dtype, mean, std, min, max, sum, and 8-element prefix/suffix fingerprint.
For integer tensors: shape, dtype, sum, and full values.

| Class | Golden file | Code under test |
|---|---|---|
| `TestPose2TextRegression` | `golden/pose2text.json` | `Pose2TextTranslationProcessor.__call__()` |
| `TestVideo2TextRegression` | `golden/video2text.json` | `Video2TextTranslationProcessor.__call__()` |
| `TestFeatures2TextRegression` | `golden/features2text.json` | `Features2TextTranslationProcessor.__call__()` |
| `TestText2TextRegression` | `golden/text2text.json` | `Text2TextTranslationProcessor.__call__()` |
| `TestImage2TextRegression` | `golden/image2text.json` | `Image2TextTranslationProcessor.__call__()` |
| `TestLabelsRegression` | `golden/labels.json` | `create_seq2seq_labels_from_samples()` |

**After refactoring:**
- Replace each `TestXxxRegression` processor class with the equivalent
  `MultimodalMetaProcessor` configuration and verify it matches the same golden file.
- Replace `TestLabelsRegression` with a test that calls
  `TextModalityProcessor(role="label").process_batch()` and verifies it matches
  `golden/labels.json`.
- The golden *files themselves do not change* — they are the ground truth.

| `TestPoseModalityProcessor` | `process_sample()` loads a `.pose` file → tensor; `process_batch()` pads and returns mask |
| `TestVideoModalityProcessor` | `process_sample()` loads a `.mp4` file → tensor; `process_batch()` pads and returns mask |
| `TestTextModalityProcessorPrompt` | `role="prompt"`: tokenises strings; returns `(ids, mask)` |
| `TestTextModalityProcessorEncoder` | `role="encoder"`: same as prompt |
| `TestTextModalityProcessorLabel` | `role="label"`: concatenates `decoder_prompt + output + EOS`; pads with `-100` |

**Relationship to regression tests:** Once implemented,
`TestTextModalityProcessorLabel` must produce output matching `golden/labels.json`.
